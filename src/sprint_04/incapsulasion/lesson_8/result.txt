Декларативное и императивное программирование — это два разных подхода к написанию кода.
 В императивном подходе программа описывает, как нужно выполнить задачу.
 В декларативном — что нужно сделать.

Сущности — это объекты, представляющие реальный мир или абстрактные понятия,
которые можно использовать в коде. У них могут быть свойства и методы. Свойства представляют
собой характеристики объекта, а методы — действия, которые умеет выполнять объект.

Пакет в Java — это папка с файлами классов. Пакеты объединяют классы по смыслу и назначению.

Чтобы взаимодействовать в коде с классами из пакетов стандартной библиотеки, их нужно импортировать.
Для этого используют ключевое слово import, имя пакета и имя класса.

Модификаторы доступа позволяют скрывать или, наоборот, делать общедоступными разные
 части программы: методы, поля и классы.

Всего в Java четыре модификатора уровня доступа. Для обозначения трёх из них в коде используются
 ключевые слова public, private, protected.
  У четвёртого нет обозначения — это модификатор по умолчанию default, или package-private.

Геттеры и сеттеры нужны для работы с полями класса, закрытыми модификатором private.
 get-методы позволяют получать значения из закрытых переменных,
  set-методы — сохранять в такие переменные новые значения.

Понимание принципов объектно-ориентированного программирования поможет разрабатывать
 более качественный код, который будет гибким, модульным и легко поддающимся сопровождению и расширению.
Знание модификаторов доступа, а также геттеров и сеттеров позволит правильно организовать
взаимодействие между разными частями программы и обеспечить конфиденциальность данных.
Понимание пакетов в Java поможет создавать более структурированный и организованный код.

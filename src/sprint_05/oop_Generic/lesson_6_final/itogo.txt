

Что вы узнали
Это была непростая тема. Но практика вам поможет освоить все тонкости параметрического полиморфизма! Вот что вы узнали:
По аналогии с обычными параметрами методы могут иметь параметры типа, которые определяют,
 с каким типом значений будет работать метод. Благодаря этому можно писать универсальные методы,
  которые работают с разными типами значений.

Стандартное имя для параметра типа — T, но оно может быть любым. Имя параметра типа указывается при объявлении
метода в угловых скобках перед типом возвращаемого значения: static <T> void printSmart.

В отличие от обычных параметров, параметр типа обычно не нужно явно передавать в метод:
 компилятор определяет его значение самостоятельно. Такой процесс называется автоматическим выводом типов.

В качестве параметра типа могут использоваться только ссылочные типы. Для примитивов нужно использовать классы-обёртки.

Параметр типа может использоваться и в аргументах метода, и в возвращаемом значении, а также в самом теле метода.
Единственное ограничение — нельзя создавать новые объекты этого типа через new.

Параметры типа можно использовать не только в отдельных методах, но и в целых классах.
Такие классы называются параметризованными классами.

 Для класса имя параметра типа указывается в угловых скобках после имени класса: Pair<T>.
Параметризованные методы и классы также называются дженериками (от англ. generics — «обобщения»).

Все коллекции в стандартной библиотеке Java представляют собой параметризованные классы и могут хранить
 объекты разных типов данных.

При использовании классов, связанных отношением наследования, в качестве параметров типов нарушается иерархия наследования:
общим классом для ArrayList<Guest> и ArrayList<VipGuest> будет класс Object.

Чтобы ограничить значения, которые может принимать параметр типа,
 наследниками определённого класса применяется верхняя граница типа: class GuestsList<T extends Guest>.
  В качестве верхней границы можно использовать не только обычные классы, но и интерфейсы.

Если нужно добавить ограничение на параметр типа, но при этом не важно, какое именно значение будет у этого параметра,
используются wildcard-символы: ArrayList<? extends T> — список объектов любого типа, который будет наследником типа T.
На этом всё: дженерики точно станут вашими постоянными спутниками в путешествии по миру Java! Именно дженерики делают этот язык таким универсальным и удобным. Но, конечно, в сочетании с другими видами полиморфизма.
